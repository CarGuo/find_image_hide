Image Forensics Inspector

本地图片隐写 / 来源凭证取证分析

推荐方式：直接把文件夹拖进来扫描

支持文件夹拖拽： 从资源管理器 / Finder 里把整个文件夹（含子目录）直接拖到下方方框， 所有图片会一次性上传到本机后端并自动开始批量扫描。 子目录结构会被保留，全部处理都在本机进行，不发往外网。
把文件夹拖到这里 / 点此选择
支持 JPG / PNG / WebP / BMP / TIFF / GIF —— 整个文件夹一次拖进来即可

方式二：填本地目录路径

如果图片体积比较大，更推荐这种方式：直接告诉后端一个本机绝对路径，省掉一次"上传到自己电脑"的过程。 所有分析在本地完成，不上传任何数据。

一键 Demo 说明

点击「一键运行 Demo」会直接分析项目内 tools/test_images 目录中预置的 25+ 张样本，覆盖：
① 正常图（JPEG / PNG）—— 零误报基准；
② 经典隐写：LSB 随机替换、LSB 文本载荷、PNG 后附 ZIP、JPEG EOI 后追加文本；
③ AI 痕迹：PNG text chunk 含 ChatGPT / DALL-E / SDXL / ComfyUI 关键字；
④ 版权 / 图库（重点强化）：Getty / Shutterstock / iStock / Unsplash / Adobe Stock / Alamy 等 5 个主流图库的可见水印示例、IPTC 完整版权字段示例（CopyrightNotice/Creator/Credit/Source/RightsUsageTerms）、 invisible-watermark DwtDct 真实嵌入示例、「AI 改图后伪造版权声明」攻击示例；
⑤ 洗图反查：pHash 命中本地参考库的同源图。
无需手填路径，全部本地处理。

版权检测：5 路交叉验证

本工具针对"版权图被洗 / AI 偷图 / 伪造版权声明"这一类问题，从 5 个角度交叉判定（详见每张图的「版权 / 图库检测」 tab）：
  ⓪ 版权字段汇总：把 EXIF Copyright/Artist + IPTC CopyrightNotice/Creator/Credit/Source/RightsUsageTerms + XMP-PLUS dc:rights / xmpRights:UsageTerms / plus:LicensorURL / Photoshop:Credit 全部摆出来，AI 伪造的版权字段一眼可辨；
  ① 元数据黑名单：30+ 主流图库 / 水印厂商关键词扫描；
  ② 可见水印 OCR：图像四角 / 中心区域 OCR 识别"烧"进像素的水印文字；
  ③ 隐形水印 (DwtDct)：用 invisible-watermark 库尝试 dwtDct / dwtDctSvd × 32/48/64 bits 解码， 匹配 SDV2/SDXL/ImaTag/ShieldMnt 等已知字典；
  ④ pHash 反查：对重压缩 / 缩放 / 轻调色鲁棒的感知哈希同源识别。

SynthID 增强（可选）

默认已内置一套纯 numpy 频域启发式探测（参考 aloshdenny/reverse-SynthID 公布的 1024×1024 低频载频网格），无需任何额外依赖即可在每张图的「AI 来源凭证」 tab 看到结果。
如果你希望使用官方 reverse-SynthID 包 + codebook 做更精细的反推，可以一键安装。 过程会执行 pip install 并下载约 220MB codebook，全部写入当前 Python 环境，仅在本机运行。 


    


    

      
注意事项 / 局限

      
    
        
  • 本工具仅提供启发式取证信号，不能证明图片绝对有/无水印或绝对是/不是 AI 生成。
    • 
        
  • 没找到 C2PA / 元数据并不代表不是 AI 图。C2PA 可能在截图、平台压缩、格式转换中丢失。
    • 
        
  • SynthID 不在常规 EXIF/XMP 中，本地无法可靠检测，需要官方验证渠道。
    • 
        
  • 所有分数都是 heuristic，不构成法律鉴定结论。
    • 
        
  • 拖拽上传仅与本机 webapp 进程通信（默认 127.0.0.1:5050），不会发送到外部服务器。但请勿把工具暴露到公网。
